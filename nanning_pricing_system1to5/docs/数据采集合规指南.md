# 数据采集合规指南

> 版本: 1.0 | 更新: 2026-05-21 | 适用范围: 南宁家政定价系统数据采集

---

## 一、法律依据

本系统数据采集严格遵守以下法律法规：

1. **《中华人民共和国网络安全法》**（2017年6月实施）
2. **《中华人民共和国数据安全法》**（2021年9月实施）
3. **《中华人民共和国个人信息保护法》**（2021年11月实施）
4. **《网络数据安全管理条例》**（2025年1月实施）
5. **《电子商务法》** — 公开商家信息的合理使用

---

## 二、可采集 vs 不可采集

### ✅ 可采集的公开数据

| 数据类型 | 范围 | 示例 |
|---------|------|------|
| 商家公开信息 | 店名、地址、营业时间 | 美团家政-XX家政 |
| 服务标价 | 公开展示的服务价格 | 日常保洁 40元/小时 |
| 服务范围 | 覆盖城区、服务品类 | 青秀区、日常保洁 |
| 商家评分 | 公开的评价星级 | 4.8分 |
| 促销信息 | 公开展示的活动价 | 限时8折 |

### ❌ 禁止采集的个人隐私数据

| 数据类型 | 原因 |
|---------|------|
| 客户姓名 | 个人信息保护法第4条 |
| 联系电话 | 属于敏感个人信息 |
| 家庭详细地址（到门牌号） | 属于行踪轨迹信息 |
| 身份证号 | 属于敏感个人信息 |
| 微信/支付宝账号 | 属于金融账户信息 |
| 评价用户头像昵称 | 属于个人信息 |
| 带有客户信息的聊天记录 | 属于私密信息 |

## 三、技术合规措施

### 3.1 请求频率控制

```python
# 每次请求间隔至少3秒
time.sleep(3)

# 每日同一平台请求上限500次
daily_limit = 500
```

### 3.2 Robots.txt 遵守

采集前检查目标网站的 `robots.txt`，禁止爬取的路径直接跳过：

```
User-agent: *
Disallow: /user/    # 用户信息页
Disallow: /order/   # 订单详情页
Disallow: /api/private/  # 私有API
```

### 3.3 数据脱敏处理

```python
# 地理信息脱敏：只保留到小区级别，不保留门牌号
address = "青秀区荣和大地"  # ✅ 合规
address = "青秀区荣和大地3栋2单元501"  # ❌ 违规
```

### 3.4 User-Agent 标识

所有HTTP请求携带合规的User-Agent，不伪装成浏览器访问：

```
Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36
```

## 四、数据存储规范

### 4.1 存储要求

| 要求 | 规范 | 原因 |
|------|------|------|
| 数据加密 | AES-256 加密存储 | 数据安全法第15条 |
| 访问控制 | 仅定价系统管理员可访问 | 最小必要原则 |
| 存储期限 | 不超过项目生命周期的2年 | 个人信息保护法第19条 |
| 数据脱敏 | 导出时自动脱敏 | 防止数据泄露 |
| 操作日志 | 所有数据导出记录留痕 | 数据安全法第29条 |

### 4.2 数据分级

| 等级 | 定义 | 示例 | 保护措施 |
|------|------|------|---------|
| L1 公开 | 对外公示信息 | 服务标价、活动价 | 基础加密 |
| L2 内部 | 企业积累数据 | 成本结构、订单统计 | 访问控制 |
| L3 敏感 | 可识别个人信息 | 手机号、详细地址 | **禁止采集** |

## 五、各平台合规特别说明

### 美团家政

- **允许**：查看并记录商家公开页面的服务项目、标价、评分
- **禁止**：翻页超过5页、采集用户评价文本、抓取商家手机号
- **建议频次**：每日各区采集1次，每次间隔>3秒

### 天鹅到家

- **允许**：查看服务品类公开价目表、覆盖城区
- **禁止**：采集阿姨个人信息、客户评价详情
- **建议频次**：每日不超过200次请求

### 58到家

- **允许**：采集公开的服务报价、商家介绍
- **禁止**：采集用户联系电话（被中间号屏蔽），采集IM聊天记录
- **建议频次**：每日不超过300次请求

### 喜事连连 / 壮家女（南宁本地平台）

- **允许**：采集门店公示价格、服务清单
- **注意**：本地平台规模较小，请求频率应更低（>5秒间隔）
- **建议频次**：每日不超过100次请求

## 六、违规风险与处置

| 风险等级 | 场景 | 后果 | 预防措施 |
|---------|------|------|---------|
| 高 | 采集用户手机号 | 行政处罚最高5000万/年营业额 | 代码层面屏蔽敏感字段 |
| 中 | 高频爬取导致业务影响 | 被目标平台投诉或封IP | 请求频率控制+代理IP池 |
| 中 | 存储未脱敏数据 | 数据泄露需强制报告 | 自动化脱敏管线 |
| 低 | 超出合理范围爬取 | 民事诉讼 | 限定爬取范围至公开报价 |

如遇目标平台明确发出停止采集通知（律师函/C&D），应立即停止相关采集活动。

## 七、数据采集伦理准则

1. **最小必要** — 只采集完成定价所需的最少字段
2. **非侵入** — 不绕开网站登录验证、不破解验证码
3. **透明** — 在内部文档中记录采集范围和用途
4. **可溯源** — 每条数据记录来源平台和采集时间
5. **动态合规** — 关注2026年数据合规法规更新
