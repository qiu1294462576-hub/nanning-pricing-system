## 一、模型架构：四层混合定价模型

采用**成本兜底+市场锚定+AI动态调价+规则校验**四层架构，确保定价在安全边际内实现利润最大化。

```
输入(订单信息)
    │
    ▼
┌─────────────────────────────────────────────────┐
│ 第一层：成本兜底层 (Cost Floor)                   │
│ 复用 CostCalculator.calculate_cost()             │
│ 输出：保本价 × 1.1 = 安全底价                     │
│ 约束：最终定价 ≥ 安全底价（绝对硬下限）              │
└─────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────┐
│ 第二层：市场锚定层 (Market Anchor)                │
│ 数据源：competitive_prices.csv                   │
│ 按(城区, 服务类型, 服务等级)聚合竞品价格区间        │
│ 输出：[竞品P25, 竞品P75] 参考区间                  │
│ 约束：最终定价 ≤ 竞品均价 × 1.2                   │
└─────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────┐
│ 第三层：AI 动态调价层 (XGBoost)                    │
│ 25维特征 → XGBoost Regressor                     │
│ 预测：最优定价倍率 = 建议价 / 保本价               │
│ 输出：AI建议加价率（如 1.45x）                    │
└─────────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────────┐
│ 第四层：规则校验层 (Rule Validation)              │
│ 区域加价(从config.yaml读取) + 季节加价 + 供需加价  │
│ 最低毛利率 ≥ 25%                                  │
│ 最终定价 = max(规则价, AI价, 安全底价)             │
└─────────────────────────────────────────────────┘
    │
    ▼
输出(最终售价 + 加价原因列表)
```

### 1.1 与旧模型的关键区别

| 维度 | 旧模型（Phase 2） | 新模型（Phase 3） | AI模型（Phase 4） |
|------|-------------------|-----------------|-----------------|

## 四、回测结论

### 4.1 总体对比

对2026年4,876笔订单进行回测（使用实际订单日期，季节乘数在对应月份正确激活）：

| 指标 | AI定价 | 规则定价 | 旧模型 |
|------|--------|---------|--------|
| 亏损率 | 0.00% | 0.00% | 46.84% |
| 平均毛利率 | **38.21%** | 37.99% | -3.45% |
| MAPE | **30.40%** | 30.56% | 32.71% |
| 准确率 | **69.75%** | 69.58% | 67.41% |
| 中位数倍率(vs保本价) | 1.558x | 1.546x | — |

> **关于亏损率**: 0% 亏损率由 `final_price = max(floor_price, rule_price, ai_price)` 中的 `floor_price = 保本价 × 1.1` 硬下限保证，是系统设计的数学恒等式，不作为模型质量度量。**MAPE、毛利率、中位数倍率才是有效度量指标。**

**关键发现**:
1. **亏损率从46.84%降到0%**：成本兜底层（保本价×1.1）彻底杜绝了亏损订单
2. **毛利率从-3.45%提升到38.21%**：修复了竞品数据偏低导致市场锚定过度压制价格的问题（原竞品数据比实际市场低28-65%，修复后差距缩小到5-20%）
3. **MAPE从32.71%降至30.40%**：AI定价比旧模型更接近市场实际成交价
4. **中位数倍率1.56x**：AI定价典型地比保本价高出56%，处于健康盈利区间
5. **市场锚定乘数从1.2x调整至1.3x**：反映竞品listing价与实际成交价的自然差距

### 4.2 分服务类型表现

| 服务类型 | MAPE | 毛利率 | 偏误 | 特点 |
|---------|------|--------|------|------|
| 日常保洁 | 26.7% | 18.9% | +20.9% | 充分竞争品类，毛利率受限，禁用区域加价 |
| 深度清洁 | **20.7%** | 26.1% | -5.8% | 表现最佳，回南天缺口仅-4.3% |
| 家电清洗 | 37.5% | 36.0% | +0.7% | 偏差最小，但家电种类差异大导致MAPE偏高 |
| 开荒保洁 | 54.3% | 29.3% | +20.9% | 缺乏面积特征，按时长定价精度有限 |

深度清洁为表现最佳品类（MAPE仅20.7%），开荒保洁因缺乏面积维度特征MAPE最高（54.3%）。

### 4.3 回南天缺口修复

| 指标 | 旧模型 | AI定价 | 改善 |
|------|--------|--------|------|
| 回南天深度清洁均价 | ¥287.4 | ¥396.3 | +37.9% |
| 与实际成交价缺口 | -30.6% | **-4.3%** | 缩小26.3pp |

**结论**: 回南天期间AI定价相比旧模型提升37.9%，与实际成交价缺口从-30.6%缩小到-4.3%（基本消除）。修复关键：竞品数据调整 + 市场锚定乘数放宽 + 按服务类型差异化毛利率约束。

## 六、定价建议

### 6.1 使用建议

1. **默认使用AI定价模式**：兼顾利润最大化与市场竞争力
2. **纯规则模式作为后备**：当AI模型不可用时自动降级
3. **回南天/三月三期间关注AI输出**：模型学习历史数据可能存在偏低倾向，必要时可手动微调
4. **定期重新训练**：建议每季度用最新订单数据重新训练，保持模型时效性

### 6.2 参数调整指南

| 参数 | 位置 | 调整影响 |
|------|------|---------|
| 安全边际(1.1) | `pricing_engine.py` 硬编码 | 调高则利润更高但可能失去订单 |
| 区域加价 | `config.yaml districts[].premium` | 影响规则层定价 |
| 季节加价 | `config.yaml season_multipliers` | 影响旺季/节假日定价 |
| 供需比阈值 | `pricing_engine.py` 第四层 | 调低则供需敏感度更高 |
| 目标毛利率下限 | `cost_parameters.csv` | 影响最低利润要求 |

### 6.3 后续优化方向

1. **引入转化率数据**：当前模型以历史成交价为训练目标（而非转化率x利润），如有转化率数据可改为收益管理优化
2. **强化学习替代XGBoost**：当前XGBoost为静态模型，后续可用stable-baselines3的PPO/DQN实现实时动态调价
3. **A/B测试框架**：在生产环境中同时运行AI定价和人工定价，对比实际转化率和利润
