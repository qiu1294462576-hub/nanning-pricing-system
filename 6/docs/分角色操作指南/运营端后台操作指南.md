## 二、成本参数管理（核心功能）

### 2.1 参数体系

所有成本参数存储在 `data/nanning/cost_parameters.csv`，共 8 大类：

| 参数类别 | 包含子项 | 举例 |
|----------|----------|------|
| 师傅到手时薪 | 新手/熟练/金牌 | 熟练=28 元/h |
| 社保系数 | 通用 | 1.20（到手工资 × 1.2 = 含社保成本） |
| 耗材费率 | 日常保洁/深度清洁/开荒保洁/家电清洗 | 日常保洁=5 元/h |
| 交通加价 | 7 个城区 | 青秀=5, 武鸣=25 |
| 时间段加价 | 工作日/周末/节假日/旺季 | 周末=1.1 |
| 管理费率 | 通用 | 0.06（6%） |
| 风险裕量 | 通用 | 0.10（10%） |
| 目标毛利率 | 最低/最高 | 最低25%, 最高35% |

### 2.2 如何修改参数

**方式一：通过后台页面修改（推荐）**

1. 进入"参数设置"页面
2. 找到需要修改的参数行，点击"编辑"
3. 输入新值，系统实时显示预览：
   ```
   ┌──────────────────────────────┐
   │  修改熟练时薪                 │
   │  当前值：28 元/小时           │
   │  新值：[____] 元/小时         │
   │                              │
   │  📊 影响预览：                │
   │  青秀日常保洁3h报价变更：      │
   │  旧：¥117 → 新：¥124 (+6.0%) │
   │  影响订单数（上周）：约 45 单  │
   └──────────────────────────────┘
   ```
4. 确认后，系统自动：
   - 备份旧参数（CSV 文件自动加时间戳备份）
   - 写入新参数
   - 重新加载定价引擎
   - 前端计算器同步更新

> ⚠️ **修改参数后系统自动重载，所有在线报价立即生效，无需重启服务。**

**方式二：通过 API 修改（批量/脚本）**

```bash
curl -X PUT http://localhost:8000/api/cost-params \
  -H "Content-Type: application/json" \
  -d '{
    "updates": [
      {"category": "师傅到手时薪", "subcategory": "熟练", "value": 30},
      {"category": "交通加价", "subcategory": "武鸣", "value": 30}
    ]
  }'
```

### 2.3 参数合理性校验

系统内置了自动校验规则，防止误操作：

| 校验项 | 合理范围 | 超出时 |
|--------|----------|--------|
| 师傅时薪 | 10-100 元/h | 拒绝 + 提示 |
| 社保系数 | 0-2.0 | 拒绝 + 提示 |
| 管理费率 | 0-2.0 | 拒绝 + 提示 |
| 风险裕量 | 0-2.0 | 拒绝 + 提示 |
| 目标毛利率 | 0-2.0 | 拒绝 + 提示 |

### 2.4 参数修改记录追溯

所有参数修改都有审计记录：
- CSV 文件自动备份（格式：`cost_parameters.csv.backup.20260526_143052`）
- 备份文件保留最近 30 天
- 可通过对比新旧 CSV 查看所有变更

## 三、定价规则调整

### 3.1 全局规则配置

规则配置在 `config.yaml` 中，包括：

- 城区加价系数（district premium）
- 季节乘数（season_multipliers）
- 供需比加价阈值
- 市场锚定乘数（MARKET_ANCHOR_MULTIPLIER = 1.3）
- 目标毛利率（TARGET_MARGIN_RATES）

### 3.2 修改规则配置

编辑 `config.yaml` 后，需要**重启后端服务**使其生效：

```bash
# 重启 uvicorn
uvicorn backend.main:app --host 0.0.0.0 --port 8000
```

持久化参数的修改无需重启（通过页面或 API 修改即可立即生效）。

### 3.3 常见运营调整场景

| 场景 | 调整什么 | 在哪改 |
|------|----------|--------|
| 师傅涨工资了 | 师傅到手时薪 | 参数设置页面 |
| 汽油涨价，交通费上调 | 交通加价 | 参数设置页面 |
| 社保基数调整 | 社保系数 | 参数设置页面 |
| 春节前想多涨价 | 季节乘数 → 春节 multiplier | config.yaml（需重启） |
| 竞争太激烈想降价 | 市场锚定乘数 → 降低 | config.yaml（需重启） |
| 想提高利润率 | 目标毛利率 | 参数设置页面 |

## 四、数据监控与分析

### 4.1 数据看板（每日必看）

进入"数据看板"页面，关注以下核心指标：

| KPI | 含义 | 健康范围 | 异常处理 |
|-----|------|----------|----------|
| 总订单数 | 当日/本周/本月 | 环比不下降>20% | 检查是否是淡季正常波动 |
| 均价 | 平均每单金额 | 不持续下降 | 检查是否促销过度或定价偏低 |
| 平均毛利率 | 公司整体盈利水平 | >25% | 低于 20% 需排查亏损品类 |
| 亏损率 | 亏损订单占比 | <5% | 超过 10% 立即排查原因 |
| 总收入/利润 | 营收和毛利总额 | 环比增长 | 异常下滑启动专题分析 |

### 4.2 趋势分析

看板提供月度趋势图，关注：
- 价格趋势：均价是否随季节合理波动
- 订单量趋势：是否出现异常下滑
- AI vs 旧定价对比：AI 定价是否显著优于旧定价

### 4.3 城区/服务类型分布

找出问题集中的维度：
- 哪个城区亏损率最高？
- 哪个服务类型毛利最低？
- 哪个等级师傅的订单最多？

## 五、订单管理与复盘

### 5.1 订单复盘（每周一次）

进入"订单复盘"页面，可以：

1. **筛选**：按城区、服务类型、等级、时间段筛选
2. **查看**：逐单查看新成本模型 vs 旧模型的利润差异
3. **聚焦亏损**：勾选"仅看亏损订单"，快速定位问题订单
4. **聚合分析**：系统自动按维度汇总亏损分布

### 5.2 亏损原因诊断

| 亏损类型 | 典型表现 | 运维动作 |
|----------|----------|----------|
| 定价偏低 | 报价 < 保本价 | 检查是否议价过度；调整该品类基准价 |
| 工时超预估 | 实际时长 > 预估 | 优化时长评估准确性；培训销售正确估算 |
| 交通成本高 | 远郊订单亏损 | 确认武鸣/邕宁加价是否足够覆盖 |
| 旺季没涨价 | 旺季订单毛利低 | 检查季节系数是否正确生效 |

### 5.3 导出报告

复盘页面支持导出为 Excel，包含 3 个 Sheet：
- 全部订单明细
- 亏损订单（红底标记）
- 亏损原因聚合统计

## 六、系统运维

### 6.1 日常检查清单

| 频率 | 检查项 | 操作 |
|------|--------|------|
| 每日 | 后端服务是否正常运行 | 访问 `http://localhost:8000/` 看是否返回 JSON |
| 每日 | 数据看板 KPI 是否正常 | 登录后台查看 |
| 每周 | 订单复盘，检查亏损率 | 订单复盘页面 |
| 每月 | 成本参数是否需要更新 | 参数设置页面 |
| 每月 | 备份数据文件 | 复制 `data/nanning/` 到备份目录 |

### 6.2 数据备份

关键文件定期备份：

```bash
# 备份整个数据目录
cp -r data/nanning/ backup/$(date +%Y%m%d)/

# 备份模型
cp -r output/models/ backup/$(date +%Y%m%d)/models/

# 备份配置
cp config.yaml backup/$(date +%Y%m%d)/
cp data/nanning/cost_parameters.csv backup/$(date +%Y%m%d)/
```

### 6.3 常见运维操作

| 操作 | 命令 |
|------|------|
| 启动服务 | `uvicorn backend.main:app --host 0.0.0.0 --port 8000` |
| 停止服务 | `Ctrl+C` 或 `kill <pid>` |
| 查看日志 | 终端直接查看 uvicorn 输出 |
| 重新训练模型 | `python scripts/train_model.py` |
| 重新生成数据 | `python scripts/generate_realistic_data.py` |
| 运行回测 | `python scripts/backtest.py` |
| 生成成本报告 | `python core/cost_calculator.py` |
